# Runtime Kernel Review-Loop Evidence

## Context

- Feature: `018-runtime-kernel`
- Review loop found one correctness issue in the new kernel slice: review-thread facts accepted an explicit `item_id` that could disagree with `thread_id`.
- Happy-path complexity assessment: fixing this at projection/policy call sites would add scattered branches. The first-principles boundary is fact identity normalization: GitHub review-thread work identity is canonical as `github-thread:<thread_id>`, and explicit `item_id` is valid only when it matches that identity.

## Fix

- Added a failing contract test for ambiguous review-thread item identity.
- Updated `ReviewThreadFact.from_runtime_fact` to fail loudly when `payload.item_id` does not match `github-thread:<thread_id>`.
