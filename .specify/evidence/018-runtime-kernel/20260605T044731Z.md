# Runtime Kernel Review Loop Evidence: Blank Durable URL

## Context

- Feature: `018-runtime-kernel`
- Review finding: whitespace-only `result_url` values were truthy and could satisfy durable side-effect completion evidence.
- Happy-path complexity assessment: adding more state-branch conditionals would make the projection path grow. The fix remains at the completion-evidence trust boundary.

## Fix

- Added a regression test for succeeded `reply_thread` and `resolve_thread` execution facts with blank `result_url`.
- Updated completion evidence filtering so item-scoped review-thread commands require a string `result_url` whose stripped value is non-empty.
- Updated command-plan and data-model contracts to define durable evidence as non-empty and non-blank.
