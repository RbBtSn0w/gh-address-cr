gh-address-cr review --help
usage: gh-address-cr review [--auto-simple] <owner/repo> <pr_number> [--input <path>|-] [--human|--machine|--lean|--summary]

High-level PR review entrypoint.

Use when you want the full PR review workflow to run automatically.
This command waits for external review findings when they are absent,
then tells you to re-run the same review command once handoff artifacts are filled.
You may still provide findings JSON explicitly via --input <path> or --input -.
Use --auto-simple for a lightweight GitHub thread-only path that does not wait for external review findings.
Use --lean or --summary to omit verbose thread body/url/reply_evidence fields.
Default output is a structured JSON summary. Use --human for narrative text.
--machine remains a compatibility alias for the default machine summary.


gh-address-cr threads --help

High-level GitHub review-thread entrypoint.

Use when only GitHub review threads need processing.
Use --lean or --summary to omit verbose thread body/url/reply_evidence fields.
Default output is a structured JSON summary. Use --human for narrative text.
--machine remains a compatibility alias for the default machine summary.
