# External Agent Telemetry Acceptance Matrix

This matrix turns the feature specification into executable acceptance evidence.
It exists because telemetry has several adjacent high-risk boundaries: safety
filters, fail-open/fail-loud split behavior, idempotence, report coverage,
statistics thresholds, and final-gate archive evidence. Every row must cite at
least one concrete automated test.

## Maintenance Rules

- Add a matrix row before adding a new telemetry risk category.
- For every new reject rule, add at least one safe near-miss acceptance test.
- For every fail-open rule, add a matching fail-loud telemetry-command test when
  the same damaged data is exposed through a public telemetry command.
- Do not cite manual review or green CI as row evidence; rows must cite
  executable tests by full unittest method name.
- If a referenced test is renamed, update this matrix in the same change. The
  matrix audit test enforces this contract.
